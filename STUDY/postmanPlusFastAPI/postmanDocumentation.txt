2. How to Use Variables
Variables in Postman use the handlebars syntax: {{variableName}}

You can use them anywhere in your requests:

URLs: {{baseUrl}}/users/{{userId}}
Headers: Authorization: Bearer {{token}}
Request Body: {"userId": "{{userId}}"}
Query Parameters: ?id={{postId}}
3. Benefits of Using Variables
✅ Environment Switching: Easily switch between development, staging, and production by changing the active environment—no need to manually update URLs in every request

✅ Reusability: Define once, use everywhere. Update the variable value in one place and all requests using it are automatically updated

✅ Collaboration: Share environments with your team so everyone uses the same configuration

✅ Security: Mark sensitive values (like API keys) as "secret" type to hide them from view

✅ Dynamic Workflows: Use variables to pass data between requests (e.g., capture an auth token from login and use it in subsequent requests)

Why Folders Are Useful for Organization
1. Logical Grouping: Folders help you group related endpoints together. In your case, all CRUD operations for posts are now in one place, making it easy to find what you need.

2. Scalability: As your API grows, you can create folders for different resources (Users, Comments, Categories, etc.). This prevents your collection from becoming a long, unmanageable list.

3. Clear Structure: Folders create a visual hierarchy that mirrors your API's structure, making it easier for team members to understand and navigate.

Powerful Folder Features
Authentication Inheritance: You can set authentication at the folder level, and all requests inside will automatically inherit it. For example, if all your Posts endpoints need an API key, you can set it once on the Posts folder instead of on each individual request.

Folder-Level Scripts:

Pre-request scripts run before every request in the folder (great for setting common variables or tokens)
Test scripts run after every request (perfect for common validations like checking response times)
Folder Variables: You can define variables scoped to just that folder, keeping your variable management organized.

Collection Runner executes multiple requests in sequence automatically. It's like pressing "Send" on all your requests at once, in order, with test validation.
