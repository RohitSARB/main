Filter by text
Locators can be filtered by text with the locator.filter() method. It will search for a particular string somewhere inside the element, possibly in a descendant element, case-insensitively. You can also pass a regular expression.

    .filter({ hasText: 'Product 2' })

Use a regular expression:

1️⃣ What is a “dialog” in Playwright?

A dialog is a browser-native popup created by JavaScript:

JS Function	Dialog Type
alert()	Alert
confirm()	Confirm
prompt()	Prompt
beforeunload	Page exit warning

These are not HTML elements, so:
❌ You cannot inspect them
❌ You cannot use locators
✅ You must handle them via Playwright events

2️⃣ Your code – line-by-line explanation
page.once("dialog", lambda dialog: dialog.accept())
page.click("#alert-btn")

Dry run:

page.once("dialog", ...)

Registers a one-time listener

Only the next dialog is handled

User action:

page.click("#alert-btn")


→ JS executes alert("...")

Playwright intercepts the dialog

dialog.accept() clicks OK

page.once("dialog", lambda dialog: dialog.dismiss())
page.click("#confirm-btn")

Dry run:

Register one-time handler

Button triggers confirm("Are you sure?")

dismiss() clicks Cancel

3️⃣ Dialog object – EVERYTHING you can do

Inside the lambda, dialog is a Dialog object.

page.once("dialog", lambda dialog: print(dialog.message))

Available properties & methods:
Method / Property	Meaning
dialog.type	"alert", "confirm", "prompt", "beforeunload"
dialog.message	Text shown in popup
dialog.accept()	Click OK
dialog.accept("text")	Fill prompt + OK
dialog.dismiss()	Click Cancel
4️⃣ Handling PROMPT dialogs (very important)
JavaScript:
prompt("Enter your name")

Playwright:
page.once(
    "dialog",
    lambda dialog: dialog.accept("Rohit")
)
page.click("#prompt-btn")


✔ Enters text
✔ Clicks OK

5️⃣ Reusable dialog handler (PRO STYLE)

Instead of repeating lambdas:

def handle_dialog(dialog):
    print("Type:", dialog.type)
    print("Message:", dialog.message)

    if dialog.type == "alert":
        dialog.accept()

    elif dialog.type == "confirm":
        dialog.dismiss()

    elif dialog.type == "prompt":
        dialog.accept("Auto response")


Usage:

page.once("dialog", handle_dialog)
page.click("#alert-btn")

6️⃣ Handling MULTIPLE dialogs (not just once)
❌ once → only one dialog
✅ on → handle all dialogs
page.on("dialog", lambda dialog: dialog.accept())


⚠️ Be careful:
This will accept every dialog, even unexpected ones.

7️⃣ Safe pattern (register BEFORE action)

❌ WRONG:

page.click("#alert-btn")
page.once("dialog", ...)


✔ RIGHT:

page.once("dialog", lambda d: d.accept())
page.click("#alert-btn")


Dialogs block JS execution — if handler isn’t ready, test hangs.

8️⃣ Waiting for dialogs explicitly
with page.expect_event("dialog") as d:
    page.click("#alert-btn")

dialog = d.value
print(dialog.message)
dialog.accept()


✔ Best for debugging
✔ Best when dialog text matters

9️⃣ Real-world test scenarios
✔ Assert dialog text

    page.click("#delete-btn")

✔ Handle beforeunload (page close warning)

## ⚠️ Issues with current approach

1. ❌ **Hard sleep (`time.sleep`)**

   * Breaks on slow networks
   * Unnecessary waiting on fast ones

2. ❌ **Blind looping (`for _ in range(5)`)**

   * Might stop too early
   * Might scroll too much

3. ❌ **No “did new content load?” check**

   * Infinite scroll is about **detecting growth**

## ✅ Best Practice #1 — Scroll until no new items load (RECOMMENDED)

### ✔ Most reliable pattern

```python
container = page.locator("#infinite")
items = container.locator(".card")

prev_count = 0

while True:
    container.evaluate("el => el.scrollTop = el.scrollHeight")

    # wait for either new items or timeout
    try:
        page.wait_for_function(
            "(container, prev) => container.querySelectorAll('.card').length > prev",
            arg=(container, prev_count),
            timeout=3000
        )
    except:
        break  # no new items loaded

    curr_count = items.count()
    if curr_count == prev_count:
        break

    prev_count = curr_count

print("Total items loaded:", prev_count)
```

✅ Scrolls **only when needed**
✅ Stops automatically
✅ No blind sleep
